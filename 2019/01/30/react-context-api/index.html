# 学习React Context API

  • 构造Context对象
import React from 'react';
import lightLogo from './../assets/medium_light.png';
import darkLogo from './../assets/medium_dark.png';
export const themeConfig = {
    light: {
        headerBg: '#F7B30C',
        fontColor: 'black',
        bodybg: 'white',
        logo: lightLogo
    },
    dark: {
        headerBg: '#3c3c3c',
        fontColor: 'white',
        bodybg: 'black',
        logo: darkLogo
    }
};
const ThemeContext = React.createContext(themeConfig.light);
export default ThemeContext;

createContext 返回一对Provider 和Comsumer 组件 themeConfig.light作为当前默认值？？

这里有一个小小的坑点；createContext()方法的参数，并不是初始值，其参数仅仅是为了在Provider不存在的情况下初始化context，就是说<ThemeContext .Comsumer>没有找到对应的Provider，那么这个Consumer的参数就是创建这个ThemeContext 时候createContext中的参数。所以真正意义上的设初始值应该是在Provider的value属性上设。英文文档这里写得比较明确 The defaultValue argument is only used when a component does not have a matching Provider above it in the tree. This can be helpful for testing components in isolation without wrapping them. Note: passing undefined as a Provider value does not cause consuming components to use defaultValue. If there is no Provider for this context above, the value argument will be equal to the defaultValue that was passed to createContext().

  • 构造Provider

import React, { Component } from 'react';
import Body from './Body.component';
import ThemeContext from './context/ThemeContext';
import {themeConfig} from './context/ThemeContext';

class App extends Component {

    state = {
      theme: 'light',
    };

  toggleTheme = event => {
    console.log(event);
    const currTheme = this.state.theme;
    this.setState({
      theme: currTheme === 'dark' ? 'light' : 'dark',
    });
  };

  render() {
    return (
      <ThemeContext.Provider value={themeConfig[this.state.theme]}}>
        <button onClick={this.toggleTheme}>click me</button>
        <Body/>
      </ThemeContext.Provider>
    );
  }
}

export default App;

Provider可以让childComsumer订阅到context的变化，他有一个value的属性，属性的值可以被所有的childComsumer获取到。

  • 构造Comsumer
import React, { Component } from 'react'
import ThemeContext from './context/ThemeContext';

export default class Body extends Component {
  render() {
    return (
      <ThemeContext.Consumer>
        {
          (theme) => (
            <div className="body" style={{color: theme.fontColor, background: theme.bodybg}}>
             Some text goes here
            </div>
          )
        }
      </ThemeContext.Consumer>
    )
  }
}

Comsumer可以获取到Context的值，在<ThemeContext.Consumer>内部，需要写一个函数，参数就是Provider的value属性传下来的值,这里写成theme,，并返回一个react node

还有一种方法可以获取到ThemeContext的value Body 的contextType 赋值为ThemeContext，在生命周期函数中用this.context可以获取到 the nearest current value of that Context type

原文这样说的：

The contextType property on a class can be assigned a Context object created by React.createContext(). This lets you consume the nearest current value of that Context type using this.context. You can reference this in any of the lifecycle methods including the render function.


 // 在Body.js文件中
class Body extends React.Component {
  static contextType = ThemeContext;
  render() {
    let theme= this.context;
    /* render something based on the value */
  }
}

// 在App.js文件中你的代码如果是这样写的。。

import React, { Component } from 'react';
import Body from './Body.component';

export const themeConfig = {
  light: {
      fontColor: 'black',
      headbg: 'white',
  },
  dark: {
      fontColor: 'red',
       headbg: 'black',
  },
};

const ThemeContext = React.createContext(themeConfig.dark);

export { ThemeContext };

// Body.contextType = ThemeContext;  在这里加上

class App extends Component {
  // here is some code
}

上述写法在Body 的render中得到this.context是空对象。因为在App中import Body时候，静态属性static contextType = ThemeContext 被赋值这句代码就已经执行了，然而这时候App.js中还没有执行到ThemeContext的赋值语句，所以ThemeContext还是undefined

解决方案：

  1. 在上述代码中加上注释掉的那句，保证Body.contextType 的赋值一定是在ThemeContext创建好以后赋值的
  2. 把ThemeContext单独放在一个文件中，如本文第一段代码，在Body中import，也能保证ThemeContext赋值在contextType 赋值之前

# 接下来看使用useContext Hook模拟Comsumer

import React, { useContext } from 'react'
import ThemeContext from './context/ThemeContext';

export const Body = () => {
  const theme = useContext(ThemeContext);
  console.log('theme', theme);
  return (
     <div className="body" style={{color: theme.fontColor, background: theme.bodybg}}>
             Some text goes here
      </div>
  );
};

这样theme获取到的值跟Consumer参数theme一样，但是这个Body仍然要包裹在ThemeContext.Provider里面，不然也不会发生变化。
