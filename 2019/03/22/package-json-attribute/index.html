# package.json 属性分析

官网地址

# name:string

包名，必须

# version:string

版本号，必须

npm规定了, 一个package是由名字加版本号作为唯一标识的.

# description:string

描述，可通过npm search 搜索到

# keywords:string[]

关键字,可通过npm search 搜索到

# private:boolean

是否私有，为true就不能发布出去

# author:string|object

作者

"author" :{ 
    "name" : "gtjll", 
    "url" : "https://github.com/gtjll"
},

# license:string

为你的模块制定一个协议，让用户知道他们有何权限来使用你的模块，以及使用该模块有哪些限制

# repository:string|object

项目源码的地址

"repository" :{ 
    "type" : "git", 
    "url" : "https://github.com/npm/npm.git"
},

# main :string

"main": "dist/index.js",

入口文件地址 当其他项目引用这个包的时候，就从dist/index.js里面找，dist下面的文件一定要build以后才有的

# dependencies：object

dependencies字段指定了项目运行所依赖的模块，格式为对象

"deependencies": {
    "browserify": "13.0.0",
    "karma-browserify": "~5.0.1"
}
  • 直接版本号就是制定版本，
  • 波浪线+指定版本，就是说安装时不改变大版本号和次要版本号，比如~1.2.2，表示安装1.2.x的最新版本（不低于1.2.2），但是不安装1.3.x
  • 插入号（^）+指定版本：就是说安装时不改变大版本号，比如^1.2.2，表示安装1.x.x的最新版本（不低于1.2.2），但是不安装2.x.x

# devDependencies:object

devDependencies指定项目开发所需要的模块，格式为对象

# peerDependencies: object

它是用于你这个包并不依赖模块A, 但是要使用你这个包, 必须当前项目装了A.

比如你的 my-project , 已经引入了people模块, 而还有一个people-fly, 必须你的my-project有用到people才可以用, 但是people-fly的源码里并没有直接使用people.

这时候, people-fly的package.json 里就会声明peerDependencies包含people

摘录自：package.json里的一些属性讲解

# scripts:object

执行脚本

 "scripts": {
    "start": "react-scripts start",
    "build": "react-scripts build",
    "test": "react-scripts test",
    "eject": "react-scripts eject"
  }

执行 yarn start 其实是执行react-scripts start

# engines:object

指明了该项目所需要的node.js版本。也可以指定适用的npm版本 node版本大于8,npm版本大于5

  "engines": {
    "node": ">=8",
    "npm": ">=5"
  },

# files:[]

files是一个数组, 它描述了你 npm publish的时候推送到npm服务器的文件列表,支持目录和通配 比如

"files": [
    "LICENSE",
    "History.md",
    "Readme.md",
    "index.js",
    "lib/"
  ],

  "files": [
    "dist"
  ]

写了一个文件夹名，则文件夹下所有的文件也会被包含进来

# 非官方字段

# module:string 属于发行打包相关字段

"module": "dist/index.es.js",

指向的应该是一个基于 ES6 模块规范的使用ES5语法书写的模块。

听起来是不是比较拗口？基于 ES6 模块规范是为了用户在使用我们的包时可以享受 Tree Shaking 带来的好处；使用 ES5 语法书写是为了用户在配置 babel 插件时可以放心的屏蔽 node_modules 目录。 当打包工具遇到我们的模块时：

如果它已经支持 pkg.module 字段则会优先使用 ES6 模块规范的版本，这样可以启用 Tree Shaking 机制。 如果它还不识别 pkg.module 字段则会使用我们已经编译成 CommonJS 规范的版本，也不会阻碍打包流程。

聊聊 package.json 文件中的 module 字段

# resolutions：object 属于yarn相关字段

指定版本

  "resolutions": {
    "@types/react": "16.8.8",
    "@types/react-dom": "16.8.3"
  },

# workspaces: string[]

 "workspaces": [
    "packages/*",
    "projects/*"
  ],

# browserslist：string[]

众所周知为兼容所有浏览器，有的CSS属性需要对不同的浏览器加上前缀。 Autoprefixer使用Browserlist来确定哪些浏览器版本将得到支持从而添加前缀。格式为数组。

  "browserslist": [
    ">0.2%", // 全球有超过0.2%的人使用的浏览器
     "last 2 versions", //根据CanIUse.com追踪的最后两个版本的所有浏览器
    "not dead",
    "not ie <= 11",// 不支持ie11及以下
    "not op_mini all"
  ]
